# DESIGN.md

本文件用于指导后续 UI 开发。项目是桌面端优先的阅读随笔与摘录管理网页，不是在线电子书阅读器。核心体验是管理书籍、记录摘录、基于摘录写随笔与札记，并用标签、书籍和时间线整理个人思想。

## 1. 产品气质

整体气质：安静、克制、有书写感的知识工作台。

设计参考的转译方式：

- Notion 的温和极简：低噪声界面、清楚的信息层级、内容优先，避免装饰压过文字。
- Mintlify 的阅读优化排版：稳定的阅读宽度、舒适行距、清晰段落节奏，长文本不疲劳。
- Linear 的精确桌面效率工具感：紧凑但不拥挤，状态反馈明确，控件位置稳定，适合反复使用。

界面不应像营销落地页、社交媒体信息流或视觉展示站。第一屏应直接进入工作台，而不是介绍产品功能。

关键词：纸面、书桌、索引、沉思、秩序、轻量工具。

## 2. 页面布局

### 框架

- 桌面端优先，推荐主工作区宽度为 `1200px` 到 `1440px`。
- 页面使用单一工作台外壳：左侧可承载导航或索引，中间为主内容，右侧为上下文信息。
- 布局应稳定，切换视图时避免大面积跳动。
- 页面区块使用全宽或自然分栏，不把大区域做成层层嵌套卡片。

### 推荐布局

- 书架页：左侧为轻导航或筛选入口，中间为书籍网格，右侧可显示最近记录、标签或统计。
- 书籍详情页：顶部显示书籍身份，主区域为摘录和随笔列表，侧边显示书籍元信息和相关标签。
- 编辑器页：中间是写作纸面，右侧是记录属性面板。专注模式可隐藏右侧面板和次要按钮。
- 时间线页：按日期组织记录，突出“什么时候产生了什么想法”，不是无限下滑的信息流。
- 标签页：以标签索引思想主题，标签详情页展示相关书籍、摘录和随笔。

### 密度

- 桌面端信息密度应接近效率工具，避免巨大的营销式留白。
- 写作区域保留更宽松的内边距，列表和工具栏保持紧凑。
- 控件高度建议：普通按钮 `32px` 到 `40px`，输入框 `36px` 到 `44px`，列表行 `48px` 到 `64px`。

### 使用规则

- 主界面最多同时出现 2 个强调色。优先使用 sage 和 clay，blue 和 gold 只用于特定语义。
- 背景可以有极轻微的层次变化，但不要使用大面积彩色渐变。
- 阴影应轻而短，表达层级，不制造漂浮感。推荐 `0 12px 32px rgba(36, 33, 28, 0.08)`。
- 所有文字与背景对比度应满足可读性，正文不使用低对比灰。

## 4. 字体与中文排版规范

### 字体

- UI 字体：`"PingFang SC", "Microsoft YaHei UI", "Microsoft YaHei", "Segoe UI", sans-serif`
- 正文长文：`"Noto Serif SC", "Source Han Serif SC", "Songti SC", STSong, serif`
- 书写感点缀：`"LXGW WenKai", "Kaiti SC", KaiTi, serif`
- 数字和时间可使用系统 UI 字体，保持表格与统计的对齐感。

### 字号与行高

- 页面标题：`28px` 到 `36px`，行高 `1.18` 到 `1.28`。
- 区块标题：`18px` 到 `22px`，行高 `1.3`。
- 列表标题：`15px` 到 `17px`，行高 `1.4`。
- 正文编辑：`17px` 到 `19px`，行高 `1.75` 到 `1.9`。
- 元信息：`12px` 到 `14px`，行高 `1.4` 到 `1.5`。
- 不用视口宽度动态缩放字体。字距保持 `0`，不要使用负字距。

### 中文排版

- 长文本最大行宽建议 `680px` 到 `760px`。
- 中文段落首行不强制缩进，使用段间距表达节奏。
- 段落间距建议为正文行高的 `0.75` 到 `1` 倍。
- 标点使用中文全角标点，英文和数字两侧不强制加空格，但同一组件内保持一致。
- 链接和标签文字不要用全大写英文风格。

## 5. 卡片规范

卡片只用于单个重复对象或明确的工具面板，例如书籍、记录、统计项、浮层。不要把整页大区块包成卡片，也不要卡片套卡片。

- 圆角：默认 `8px`，纸面可为 `6px`，小控件为 `6px` 到 `8px`。
- 边框：默认 `1px solid var(--color-border)`。
- 背景：使用 `--color-surface` 或 `--color-paper`，不要使用复杂玻璃拟态。
- 内边距：紧凑卡片 `12px` 到 `16px`，书籍卡片 `16px` 到 `20px`，编辑器纸面 `40px` 到 `64px`。
- Hover：只改变边框、背景和轻微位移。位移不超过 `1px`。
- 选中：使用左侧细线、边框加强或淡色底，不使用强烈描边和大面积高饱和色。

书籍卡片应像“索引中的一本书”，不是电商商品卡。优先展示书名、作者、记录数、最近更新时间和主题标签。封面图可以存在，但不应压过文字信息。

## 6. 组件规范

### 按钮

- Primary：用于创建记录、保存、确认等主操作。颜色使用 sage 或深色中性，不使用亮色渐变。
- Secondary：用于预览、导出、筛选等次级操作。白底或浅底，边框清楚。
- Ghost：用于返回、关闭、更多等低权重操作。仅在 hover 时显出底色。
- Icon Button：工具栏、主题切换、编辑操作优先使用图标按钮，并提供清晰 `title` 或可访问标签。
- Danger：删除操作使用 danger 色文字或边框，只有确认态才使用 danger 背景。

按钮文字应短，避免把说明写进按钮。按钮宽度稳定，hover 不应导致布局变化。

### 输入框

- 输入框背景接近纸面，边框低对比，focus 时使用明确焦点环。
- 搜索框应支持快速清空，视觉上像工具，不像宣传页订阅输入框。
- 表单标签放在输入上方或左侧，元信息区可使用紧凑纵向布局。
- 占位文字只提示格式或示例，不承载主要说明。

### 标签与 Chips

- 标签用于主题、关键词和筛选条件。
- 默认状态低饱和浅底，文字清楚。
- 选中状态可使用淡 sage 底和强边框。
- 标签不应做成彩虹色集合。每个页面中高亮色数量要克制。
- 可删除标签使用小型关闭按钮，点击区域不小于 `24px`。

### 侧边栏

- 侧边栏是上下文索引，不是广告区。
- 宽度建议 `280px` 到 `340px`。
- 内容顺序：当前上下文、筛选或元信息、最近活动、辅助入口。
- 侧边栏可滚动，但主编辑区滚动不应带动整个页面失控。
- 当前项使用细线、淡底或字体权重表达，不使用强烈色块。

### 编辑器

- 编辑器是核心，视觉权重最高。
- 写作纸面应居中，有明确最大宽度和舒适内边距。
- 摘录区和随笔区要有区分：摘录更像引用，随笔更像正文。
- 正文行高 `1.75` 到 `1.9`，段落间距清楚。
- 富文本工具条应轻、近、短。只在选区或编辑状态中出现。
- 专注模式隐藏侧边栏、删除、导出等次要项，只保留返回、保存状态和退出专注。
- 保存状态应低干扰，例如“已保存”“保存中”“离线暂存”，不要弹出强提示。

### 列表与时间线

- 记录列表应适合扫描：标题、摘要、关联书籍、标签、时间。
- 列表行 hover 显示可操作入口，默认不堆满按钮。
- 时间线按日期分组，强调思想生成的节奏，不做社交动态流。
- 空列表提供一个明确主操作，例如“新建第一条摘录”。

### 浮层、Dialog 与 Toast

- Dialog 用于预览、确认删除、导入书籍等需要打断的动作。
- Toast 用于保存、导出、复制等轻反馈。出现时间短，不遮挡编辑器正文。
- 浮层边框和阴影要明确，但不要透明玻璃化。

## 7. 状态规范

### Hover

- Hover 表示“可操作”，不能像装饰动画。
- 允许变化：背景变浅或变深、边框加强、轻微上移 `1px`、显示次要操作。
- 不允许变化：大幅缩放、旋转、发光、布局尺寸变化。

### Selected

- Selected 表示当前上下文，例如当前书籍、当前标签、当前记录。
- 推荐表现：左侧 `2px` 色条、淡色背景、字体权重 `600`、边框加强。
- 不要使用强烈霓虹色、厚重阴影或全彩背景。

### Editing

- Editing 状态应让用户知道焦点在哪里。
- 编辑器 focus 使用淡色边框或纸面阴影，不能打断写作。
- 工具条只在需要时出现，失焦后自然收起。
- 输入中可以显示“保存中”，停止输入后回到“已保存”。

### Empty

- Empty 状态要短、安静、可行动。
- 文案说明当前没有内容，并提供一个主操作。
- 不使用大插画、大面积空态营销文案或过度情绪化表达。

### Loading

- 加载状态使用骨架屏或小型进度提示。
- 列表加载保持原有布局高度，避免内容突然跳动。
- 编辑器保存不使用全屏 loading。

### Disabled

- Disabled 状态降低透明度或对比，但文字仍可读。
- 禁用原因如果不明显，应在 tooltip 或辅助文本中说明。

### Error

- 错误提示使用 danger 色，但保持克制。
- 表单错误显示在字段附近。
- 存储失败、导入失败等系统错误应提供恢复建议。

## 8. 动效与交互

- 动效时长建议 `120ms` 到 `180ms`。
- 动效曲线使用 `ease` 或轻微 `cubic-bezier(0.2, 0, 0, 1)`。
- 工具类界面动效用于反馈，不用于表演。
- 尊重 `prefers-reduced-motion`，减少位移和闪烁。
- 键盘操作要完整：Tab 顺序清楚，焦点可见，常用编辑按钮支持快捷键提示。

## 9. 明确不要做

- 不要使用过度渐变。
- 不要使用强烈霓虹色。
- 不要使用复杂玻璃拟态、厚重模糊背景或彩色光斑。
- 不要使用社交媒体风格信息流。
- 不要做营销落地页式 hero。
- 不要把所有内容装进圆角大卡片。
- 不要用巨大封面、夸张插画或装饰图压过文字。
- 不要使用一整套高饱和标签颜色。
- 不要让 hover、selected 或 loading 改变布局尺寸。

## 10. 后续 UI 开发检查清单

每次 UI 改动前检查：

- 当前改动是否服务于书籍、摘录、随笔、标签或时间线组织。
- 桌面端是否比移动端更高效，移动端是否仍可正常使用。
- 主文字是否清楚，长文是否有舒适行宽和行高。
- 控件是否有 hover、focus、selected、disabled、empty 和 loading 状态。
- 是否复用色彩 tokens，而不是随手新增相近颜色。
- 是否避免卡片套卡片、过度渐变、霓虹色和社交流风格。
- 是否让编辑器保持最高视觉优先级。
