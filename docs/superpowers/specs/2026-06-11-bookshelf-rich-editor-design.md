# 书架入口与富文本编辑器改版设计

## 背景

当前网站已经具备本地保存、书籍/主题归类、摘录与随笔编辑、搜索、预览和导出能力，但信息架构仍以“打开即编辑”为主。用户希望网站进入后先呈现书架，点进一本书后先看到这本书的记录列表，再点某条记录进入单独编辑页。同时，编辑器需要从普通文本框升级为更接近写作软件的富文本编辑器，选中文字时显示悬浮格式工具条。

## 目标

- 首页改为书架视图，首屏聚焦书籍，而不是单条笔记编辑器。
- 书籍详情页先展示该书相关的随笔、摘录、感想等记录列表。
- 单独编辑页提供富文本编辑体验，选中文本后出现悬浮编辑框。
- 优化编辑时响应速度，减少输入过程中的整页重绘和无必要保存。
- 保留当前静态站点形态，不引入构建流程，不依赖后端服务。

## 导航结构

网站使用三层视图：

1. `shelf`：书架首页
   - 展示所有书籍卡片。
   - 支持搜索书名、作者、记录内容和主题。
   - 显示轻量统计，例如书籍数量、记录数量、总字数。
   - 点击书籍进入书籍详情页。

2. `book`：书籍详情列表页
   - 顶部显示书名、作者、记录数量和返回书架入口。
   - 显示该书下的记录列表。
   - 支持按“全部 / 摘录 / 随笔 / 感想”筛选。
   - 提供“新建记录”按钮，新建后进入编辑页。
   - 点击某条记录进入单独编辑页。

3. `editor`：单条记录编辑页
   - 顶部显示返回书籍详情、保存状态、预览、导出、删除。
   - 主体为标题、摘录区、正文富文本区。
   - 右侧或窄屏下方显示元信息：类型、关联书籍、页码/章节、主题、标签。

## 富文本编辑器

编辑器使用浏览器原生 `contenteditable` 实现轻量富文本，不引入大型编辑器依赖。

### 编辑区域

- 标题继续使用普通输入框，保证稳定和易保存。
- 摘录区和正文区改为可编辑富文本容器。
- 内容保存为安全的 HTML 字符串，同时导出时转换为可读 Markdown。
- 空内容显示低干扰占位提示。

### 悬浮工具条

当用户在摘录区或正文区选中文字时，显示一个跟随选区的悬浮工具条。

工具条包含：

- 加粗
- 斜体
- 标题
- 引用
- 无序列表
- 清除格式

工具条行为：

- 选区为空时隐藏。
- 滚动、点击编辑器外部或按 Esc 时隐藏。
- 点击工具按钮后保留编辑器焦点，并立即保存内容。
- 移动端如果选区坐标不可用，则显示在编辑器顶部附近。

## 数据模型

沿用当前 `books`、`themes`、`notes` 结构，并补充视图状态：

- `activeView`：当前视图，值为 `shelf`、`book` 或 `editor`。
- `activeBookId`：当前进入的书籍。
- `activeNoteId`：当前编辑的记录。
- `bookFilter`：书籍详情页中的记录类型筛选。
- `query`：全站搜索关键词。

记录内容兼容旧数据：

- 旧字段 `quote`、`body` 继续保留。
- 新增或复用富文本字段时，旧纯文本会在首次渲染时转义并换行显示，避免破坏既有本地数据。
- 导出 Markdown 时优先使用富文本内容转换后的文本结构；没有富文本内容时回退到旧字段。

## 性能优化

当前输入时会触发较多派生区域刷新。改版后采用局部更新：

- 编辑器输入只更新当前记录、字数、保存状态和必要的 chips。
- 书架、书籍列表、主题列表只在切换视图、筛选、新建、删除或元信息改变时重新渲染。
- 自动保存继续使用防抖，输入中显示“保存中”，停止输入后再写入 `localStorage`。
- 书架和记录列表使用一次性 `DocumentFragment` 渲染，减少 DOM 插入次数。
- 搜索输入防抖过滤列表，避免每个字符都立即重建整个页面。

## 页面布局

视觉方向保留当前安静书房气质，但让入口更像“藏书架”：

- 首页卡片更像书脊/书册，显示书名、作者、记录数量、最近更新时间。
- 书籍详情页用紧凑列表展示记录，方便扫描。
- 编辑页减少装饰，重点放在写作纸面和浮动工具条。
- 窄屏下保持单列：书架卡片、书籍记录列表、编辑器依次自然铺开。

## 错误处理

- 本地存储不可用时，显示提示，但允许临时编辑。
- 解析旧数据失败时，保存损坏数据备份并回退到示例数据。
- 删除记录前保留确认。
- 富文本格式命令失败时不打断输入，只显示轻量提示。

## 测试与验收

需要验证：

- 打开网站先看到书架，而不是编辑器。
- 点击一本书后看到该书记录列表。
- 记录列表可以按全部、摘录、随笔、感想筛选。
- 点击记录进入单独编辑页。
- 新建记录会归入当前书籍，并进入编辑页。
- 选中文字后出现悬浮编辑框。
- 加粗、斜体、标题、引用、列表、清除格式可用。
- 编辑内容刷新后仍保留。
- 搜索和筛选响应明显顺滑。
- 现有测试通过，且补充纯逻辑测试覆盖视图筛选和富文本导出。

## 不在本次范围

- 用户登录和云同步。
- ISBN 查询和外部书籍数据库。
- 多设备同步。
- AI 总结或自动分类。
- 完整协同编辑。
- 引入 React、Vue 或大型富文本编辑器。
