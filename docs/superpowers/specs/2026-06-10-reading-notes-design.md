# 阅读随笔记录网页设计说明

## 背景

项目当前为空目录。用户希望开发一个以桌面端为主的阅读记录网页，核心不是书单管理，而是围绕书籍和主题写摘录、随笔、感想。用户提供了 `C:\Users\TianLutao\Downloads\reading_notes_ui_v2_elegant.html` 作为界面框架参考，页面名称为“墨读札记 V2｜极简书房风阅读随笔 UI”。

## 目标

基于参考 HTML 的三栏书房式界面，开发一个可实际使用的阅读随笔记录网页：

- 可以新建、编辑、保存阅读记录。
- 每条记录可以关联一本书和一个或多个主题。
- 记录内容包含摘录原文、随笔正文、感想或复盘。
- 可以按书籍、主题、记录类型和关键词查找。
- 首版以桌面端体验为主，同时保留基本窄屏适配。

## 实现路径比较

### 方案 A：单页 HTML + CSS + JavaScript

优点是启动最快，不需要安装框架，直接打开文件即可使用。适合当前空项目和个人工具的首版。缺点是当功能变多时，代码需要认真拆分，否则会变得拥挤。

### 方案 B：React 或 Vue 小应用

优点是组件化清晰，适合后续做复杂筛选、状态管理和多页面。缺点是需要构建工具和依赖，首版成本更高。

### 方案 C：带后端和数据库

优点是数据更稳，后续可支持多设备同步。缺点是开发范围明显扩大，当前阶段不必要。

推荐方案 A。先做成可靠的本地单页应用，数据存在浏览器本地。等使用流程稳定后，再决定是否升级为框架应用或后端存储。

## 页面框架

沿用参考界面的三栏结构：

### 左侧：书籍与索引栏

左侧负责进入和定位：

- 品牌区：保留“墨读札记”或后续替换为用户指定名称。
- 新建记录按钮：创建一条新的摘录、随笔或感想。
- 搜索框：搜索书名、摘录、正文、主题和标签。
- 导航入口：今日书写、全部随笔、摘录卡片、主题索引。
- 书籍列表：展示正在阅读或最近记录的书，每本书显示记录数量和阅读进度。

### 中间：写作台

中间是主要工作区：

- 顶部显示当前位置、保存状态、预览和导出入口。
- 稿纸区保留参考界面的沉浸写作样式。
- 字段包括标题、摘录原文、正文、页码或章节、关联书籍、关联主题、记录类型、标签。
- 正文支持普通文本首版输入，格式按钮首版做轻量交互或作为预留。

### 右侧：上下文面板

右侧帮助用户回看和整理：

- 本周积累：记录数量、字数、涉及书籍数。
- 主题连接：当前记录关联的主题。
- 可能相关：同书或同主题的其他记录。
- 今日提示：提供写作提醒，不依赖外部服务。

## 数据模型

首版使用浏览器本地存储，核心数据结构包括：

- `books`：书籍列表，包含 id、书名、作者、分类、阅读进度。
- `themes`：主题列表，包含 id、名称、颜色。
- `notes`：记录列表，包含 id、类型、标题、摘录、正文、书籍 id、主题 id 列表、页码、标签、创建时间、更新时间。

记录类型固定为：

- 摘录
- 随笔
- 感想

## 数据流

- 页面加载时读取本地存储，没有数据则使用示例数据初始化。
- 用户选择书籍、主题或导航入口时，筛选左侧和右侧展示内容。
- 用户编辑标题、摘录、正文或元信息时，自动保存到本地存储。
- 搜索时在书籍、主题、标题、摘录、正文、标签中匹配。
- 导出时生成当前记录的 Markdown 文本。

## 错误处理

- 本地存储不可用时，页面提示“当前浏览器无法保存记录”，但仍允许临时编辑。
- 解析本地数据失败时，保留损坏数据备份键，并回退到示例数据。
- 删除记录前需要确认，避免误删。
- 空记录在离开前保留草稿，不自动丢弃。

## 测试与验收

首版验收重点：

- 能新建记录并自动保存。
- 刷新页面后记录仍存在。
- 能按书籍筛选记录。
- 能按主题筛选记录。
- 搜索能命中标题、摘录、正文和标签。
- 桌面端三栏布局稳定，不出现文字挤压或主要内容重叠。
- 窄屏下右侧面板可隐藏，主写作区可正常使用。

## 不在首版范围

- 用户登录和云同步。
- 多设备同步。
- AI 总结或自动分类。
- 完整富文本编辑器。
- 复杂统计仪表盘。
- 图书 ISBN 查询或外部书籍数据库。
