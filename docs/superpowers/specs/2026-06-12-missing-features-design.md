# 缺失功能完善设计

## 背景

墨读札记是一个桌面端优先的阅读随笔与摘录管理网页。当前主界面已经具备三栏知识工作台、书籍列表、摘录/随笔列表、详情区和独立编辑器，但 README 自查中仍有若干长期使用所需的管理能力缺口。

本次设计目标是补齐这些缺口，让应用从“可以记录摘录与随笔”进一步变成“可以长期管理阅读资料和思想索引”的本地知识工作台。实现仍基于原生 HTML、CSS、JavaScript 和浏览器 localStorage，不引入后端、框架或复杂构建工具。

## 目标

- 支持完整的书籍新建、编辑、状态维护和安全删除。
- 支持摘录与随笔收藏，并提供独立收藏入口。
- 将“最近阅读”升级为按日期分组的时间线视图。
- 支持标签详情视图，聚合相关书籍、摘录和随笔。
- 支持完整数据 JSON 导出与导入恢复。
- 增强搜索与筛选，提供清空、排序、类型和状态组合条件。
- 保持 DESIGN.md 定义的安静、克制、桌面优先三栏工作台体验。

## 非目标

- 不实现在线账号、云同步、多设备冲突合并或协作功能。
- 不接入 ISBN、豆瓣、Open Library 等外部元数据服务。
- 不引入富文本历史版本、撤销栈或复杂版本管理。
- 不把应用改造成在线电子书阅读器。
- 不重构为 React、Vue、Next.js 或其他框架。

## 用户流程

### 书籍管理

用户可以在左侧或中间主区域通过“新建书籍”打开轻量弹窗，填写书名、作者、阅读状态和可选封面。创建后书籍出现在书架列表，并自动成为当前选中书籍。

用户可以在右侧书籍详情区编辑当前书籍的书名、作者、阅读状态和封面。状态包括“正在读”“已读”“待读”，并立即影响中间区域筛选。

用户可以删除没有关联记录的书籍。若书籍下已有摘录或随笔，删除按钮禁用或提示需要先迁移/删除记录，避免误删关联数据。

### 收藏

摘录和随笔卡片、右侧选中记录、编辑器顶部都提供收藏切换。收藏状态以克制图标或文字呈现，不使用高饱和色块。

左侧导航增加“收藏”。进入后，中间列表展示所有已收藏摘录与随笔，右侧详情区显示当前收藏记录及其关联书籍、标签和编辑入口。

### 时间线

“最近阅读”升级为时间线视图。中间区域按本地日期分组，展示当天新增或更新的记录。每条记录显示标题、类型、关联书籍、摘要、标签和更新时间。

时间线不是社交信息流，不做无限加载式动态流。当前数据量较小时一次性展示即可，空状态提供“新建第一条摘录”的入口。

### 标签详情

标签页显示主题和普通标签索引。点击标签后进入标签详情上下文，保留三栏布局：

- 中间区域展示该标签相关的书籍、摘录和随笔。
- 右侧详情区展示标签名称、类型、相关数量、关联书籍和最近记录。
- 搜索框在标签上下文中继续生效，用于进一步缩小结果。

主题和普通标签都使用同一套详情视图。主题来自 `themes`，普通标签来自记录 metadata 中的 tags。

### 全量导入与导出

提供“数据”或“备份”入口，放在左侧底部或右侧详情辅助区。用户可以：

- 导出全部数据为 JSON 文件。
- 从 JSON 文件导入并替换当前本地数据。

导入时先解析 JSON 并通过现有 `migrateState` 正规化。成功后刷新界面并保存。失败时给出轻量错误提示，不覆盖当前数据。

导入采用“替换当前数据”策略，不做复杂合并。导入前提示用户可先导出备份。

### 搜索与筛选

顶部搜索框增加清空按钮。搜索继续覆盖书名、作者、记录标题、正文、摘录、主题和标签。

中间区域增加排序与组合筛选：

- 书架：状态筛选 + 排序，排序支持最近更新、书名、记录数。
- 记录列表：类型筛选 + 排序，排序支持最近更新、最早创建、标题。
- 收藏和时间线：保留类型筛选与搜索。

筛选控件应紧凑，符合 DESIGN.md 中效率工具的密度，不做复杂高级搜索面板。

## 数据设计

现有 schemaVersion 维持为 2，尽量用兼容字段扩展，避免破坏旧数据。

### Book

新增或补齐字段：

- `status`: `reading | finished | planned`
- `createdAt`: ISO 字符串，可在迁移时补齐
- `updatedAt`: ISO 字符串，可在迁移时补齐

现有字段继续保留：

- `id`
- `title`
- `author`
- `coverImage`
- `coverColor`

### Note

新增或补齐收藏字段：

- `metadata.favorite`: boolean

兼容旧字段时允许读取 `note.favorite`，迁移后写入 `metadata.favorite`，并可同步 legacy alias。

### UI State

新增运行时 UI 字段：

- `workspaceSection`: 增加 `favorites`
- `selectedTag`: `{ kind: "theme" | "tag", id: string, name: string } | null`
- `sortMode`: 当前列表排序方式
- `noteTypeFilter`: `all | quote | essay`

这些字段主要用于界面状态，不作为长期业务数据的核心。

## 界面架构

继续保留当前三栏结构：

- 左侧：产品名、我的书架、全部摘录、随笔、收藏、标签、最近阅读、数据备份入口。
- 中间：搜索、筛选、排序、书籍/记录/标签/时间线列表。
- 右侧：当前书籍、当前记录、标签详情、备份操作或编辑入口。

新增 UI 优先以函数组件式 DOM builder 实现，沿用现有 `createWorkbenchSidebar`、`createWorkbenchMain`、`createWorkbenchDetail` 等模式。若单个函数明显过长，增加聚焦的小函数，而不是把所有新逻辑写进一个分支。

## 状态与反馈

- Hover：只做背景、边框或轻微位移变化。
- Selected：使用淡底、边框加强或左侧细线。
- Editing：书籍编辑和数据导入使用弹窗或右侧表单，焦点环清晰。
- Empty：每个空列表都提供一个明确主操作。
- Loading：本地操作通常无需 loading；文件导入时可显示短暂状态文字。
- Error：导入失败、删除受限、JSON 格式错误使用 toast 或字段附近提示。

## 错误处理

- 导入 JSON 解析失败：不修改现有 localStorage，提示“导入失败，请检查文件格式”。
- 导入结构不完整：尝试通过 `migrateState` 修复，仍无法得到有效数据时失败。
- 删除有关联记录的书籍：阻止删除，并提示先处理关联摘录和随笔。
- 文件读取失败：提示重新选择文件。
- 保存失败：保留现有内存状态，并提示浏览器存储可能不可用。

## 测试与验证

完成实现后运行：

```bash
node --check assets/app.js
node tests/app.test.cjs
```

项目没有 `package.json`，因此不运行 `npm run build`、`npm run lint` 或 `npm test`。不为本次功能强行新增复杂工具链。

需要补充或更新的测试重点：

- 迁移时补齐书籍时间字段和 note favorite 字段。
- 收藏筛选只返回已收藏记录。
- 时间线按本地日期分组。
- 标签详情能同时匹配主题和普通标签。
- 全量导入前会先 migrate/normalize。

## 交付标准

- README 自查中的主要功能缺口都有可见入口和可用流程。
- 新增功能在桌面三栏工作台中完成，不改变项目定位。
- 不引入强烈渐变、霓虹色、复杂玻璃拟态或社交媒体风格信息流。
- 旧 localStorage 数据可以迁移并继续使用。
- 现有测试通过，新增测试覆盖核心数据逻辑。
- 完成后提交 git commit。
