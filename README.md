# 墨读札记

墨读札记是一个桌面端优先的阅读随笔与摘录管理网页。它不是在线电子书阅读器，而是一个安静的知识工作台：用来管理书籍、记录摘录、基于摘录写随笔或感想，并通过书籍、标签和时间线整理自己的思想。

当前项目已升级为本地 Web 应用：前端使用 Vite、React 和 Tiptap，后端使用 Node.js、Express 和 SQLite。数据默认保存在本地 `data/reading-notes.sqlite` 中，旧版浏览器 `localStorage` 数据可从页面顶部提示导入。

## 当前界面

主界面采用三栏工作台：

- 左侧：产品名与导航，包括我的书架、全部摘录、随笔、标签、最近阅读。
- 中间：搜索栏、书籍状态筛选，以及书籍、摘录、随笔或标签列表。
- 右侧：当前书籍信息、选中记录、摘录列表、随笔编辑入口和关联标签。

深度编辑仍使用独立编辑器页面，保留标题、摘录、正文、元信息、预览、导出、删除、专注模式等入口。

## 本地运行

需要 Node.js 24 或更新版本。首次运行前安装依赖：

```bash
npm install
```

启动本地应用：

```bash
npm run dev
```

默认访问 `http://127.0.0.1:5173/`。

构建生产资源：

```bash
npm run build
```

运行测试：

```bash
npm test
```

## 可用功能

- 浏览书架中的书籍卡片。
- 新建书籍，并编辑书名、作者、阅读状态和封面。
- 删除没有关联记录的书籍；已有摘录或随笔的书籍会被阻止删除。
- 按全部、正在读、已读、待读筛选书籍。
- 在书籍详情区直接修改阅读状态。
- 搜索书名、作者、摘录、随笔和标签。
- 清空搜索，并按最近更新、标题、书名或记录数排序。
- 查看全部摘录。
- 查看随笔与感想。
- 收藏或取消收藏摘录和随笔，并在“收藏”入口集中查看。
- 查看标签索引。
- 点击标签或主题进入标签详情，查看相关书籍、摘录和随笔。
- 查看按日期分组的最近阅读时间线。
- 在右侧详情区查看当前书籍、摘录列表、随笔入口和关联标签。
- 点击记录进入完整 Tiptap 编辑器。
- 编辑记录标题、摘录、正文、类型、关联书籍、页码、主题和标签。
- 编辑器支持标题、加粗、斜体、下划线、删除线、列表、引用、链接、图片 URL、表格、行内代码、代码块、分割线、撤销、重做和清除格式。
- 预览并导出单条记录为 Markdown。
- 导出全部数据为 JSON 备份。
- 从旧版 `localStorage` 或 JSON 数据导入到 SQLite；导入失败时不会覆盖当前数据。
- 更换书籍封面。
- 切换明暗主题。

## 已检验的正常操作流程

本次检查基于本地预览地址 `http://127.0.0.1:5173/`。

已通过的流程：

1. 打开页面后进入“我的书架”工作台。
2. 左侧导航可切换到“全部摘录”“随笔”“标签”“最近阅读”。
3. 书架默认显示 4 本示例书，状态筛选和排序控件可见。
4. 可以新建一本测试书，并设置为“待读”。
5. 可以在右侧详情区把测试书状态改为“已读”。
6. 没有关联记录的测试书可以删除，删除后书架恢复为 4 本书。
7. 有关联摘录的书籍会阻止删除，并显示提示。
8. 可以收藏一条摘录，并在“收藏”入口看到该记录。
9. 取消收藏后，“收藏”入口显示空状态。
10. “标签”页能展示主题和普通标签索引。
11. 点击“自我整理”主题后进入标签详情，能显示 2 条相关记录、2 本相关书籍和右侧聚合信息。
12. “最近阅读”按日期分组展示记录。
13. “导出备份”能下载完整 JSON 文件。
14. 导入错误 JSON 时显示失败提示，且现有 4 本书、3 条记录没有被覆盖。
15. 搜索框、清空按钮、类型筛选和排序控件都能正常显示。

命令检查：

```bash
npm test
npm run build
```

结果：现有旧数据兼容测试和新增后端 API 测试全部通过，生产构建通过。构建阶段会提示 Tiptap/ProseMirror 包体积较大，这是成熟编辑器底座带来的正常成本，后续可以按需代码分割。

## 自查：还缺哪些功能

以下是当前版本仍缺少或不完整的功能：

- 还没有 ISBN、豆瓣、Open Library 等外部书籍元数据导入。
- 还没有批量导入书籍清单或从表格导入书籍。
- 标签本身还不能重命名、合并或删除；目前标签主要来自记录元信息。
- JSON 导入采用“替换当前数据”策略，还没有复杂合并、去重或冲突处理。
- 搜索仍是本地文本匹配，没有布尔条件、保存筛选器或全文索引。
- 移动端只是兜底适配：当前设计目标是桌面端知识工作台，移动端体验还没有作为主要流程细化。
- 自动保存状态可以继续增强：当前编辑器有保存提示和失败状态，但还没有复杂冲突处理或历史版本。
- 还没有云同步、账号、多设备备份或协作能力。
