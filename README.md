# 墨读札记

墨读札记是一个桌面端优先的阅读随笔与摘录管理网页。它不是在线电子书阅读器，而是一个安静的知识工作台：用来管理书籍、记录摘录、基于摘录写随笔或感想，并通过书籍、标签和时间线整理自己的思想。

当前项目使用原生 HTML、CSS 和 JavaScript 实现，数据保存在浏览器 `localStorage` 中。

## 当前界面

主界面采用三栏工作台：

- 左侧：产品名与导航，包括我的书架、全部摘录、随笔、标签、最近阅读。
- 中间：搜索栏、书籍状态筛选，以及书籍、摘录、随笔或标签列表。
- 右侧：当前书籍信息、选中记录、摘录列表、随笔编辑入口和关联标签。

深度编辑仍使用独立编辑器页面，保留标题、摘录、正文、元信息、预览、导出、删除、专注模式等入口。

## 本地运行

这个项目没有构建步骤，可以直接打开 `index.html`。

为了避免浏览器对本地文件的限制，也可以用任意静态服务启动项目目录。例如：

```bash
npx serve .
```

如果没有安装额外工具，也可以使用任何能把当前目录作为静态站点的本地服务。启动后访问对应的本地地址即可。

## 可用功能

- 浏览书架中的书籍卡片。
- 按全部、正在读、已读、待读筛选书籍。
- 搜索书名、作者、摘录、随笔和标签。
- 查看全部摘录。
- 查看随笔与感想。
- 查看标签索引。
- 查看最近阅读记录。
- 在右侧详情区查看当前书籍、摘录列表、随笔入口和关联标签。
- 点击记录进入完整编辑器。
- 编辑记录标题、摘录、正文、类型、关联书籍、页码、主题和标签。
- 预览并导出单条记录为 Markdown。
- 更换书籍封面。
- 切换明暗主题。

## 已检验的正常操作流程

本次检查基于本地预览地址 `http://127.0.0.1:4173/`。

已通过的流程：

1. 打开页面后进入“我的书架”工作台。
2. 左侧导航可切换到“全部摘录”“随笔”“标签”“最近阅读”。
3. “全部摘录”能显示 2 条示例摘录。

8. 点击《设计心理学》后，右侧详情区能更新为该书信息，并显示选中随笔、随笔入口和关联标签。
9. 从随笔卡片点击“编辑”后，可以进入完整编辑器。
10. 编辑器中标题、摘录编辑区、正文编辑区、元信息侧栏、预览和导出入口都存在。
11. 检查期间浏览器控制台没有发现错误。

命令检查：

```bash
node --check assets/app.js
node tests/app.test.cjs
```

结果：现有 16 个 Node 测试全部通过。

## 自查：还缺哪些功能

以下是当前版本仍缺少或不完整的功能：

- 书籍导入/新建还不完整：目前有示例书籍和更换封面入口，但没有正式的新建书籍、批量导入书籍或 ISBN/元数据导入流程。
- 书籍状态管理缺失：界面有“正在读、已读、待读”筛选，但还没有修改书籍状态的表单。旧本地数据会默认归为“正在读”，所以“已读”“待读”可能没有结果。
- 收藏功能未实现：当前左侧没有“收藏”入口，也没有收藏摘录或随笔的能力。
- 时间线仍较轻量：目前“最近阅读”只是最近记录列表，还没有按日期分组的真正时间线视图。
- 标签详情页不完整：标签可以展示和搜索，但点击标签后的深度筛选、标签详情统计和相关书籍聚合还比较初步。
- 全局数据导入/导出缺失：单条记录可以导出 Markdown，但还没有完整备份、恢复或迁移全部数据的功能。
- 书籍详情编辑不足：书名、作者、状态等书籍元信息暂时不能在界面中直接编辑。
- 搜索仍是简单文本匹配：没有高级筛选、组合条件、排序方式或清空按钮。
- 移动端只是兜底适配：当前设计目标是桌面端知识工作台，移动端体验还没有作为主要流程细化。
- 自动保存状态可以继续增强：现有编辑器有保存提示，但还没有更完整的保存失败恢复、冲突处理或历史版本。

## 后续开发建议

后续 UI 修改前请先阅读根目录 `DESIGN.md`，并遵守 `AGENTS.md` 中的开发规则。优先补齐书籍管理、状态编辑、收藏、时间线和完整数据导入/导出，这些功能会让当前工作台从“可记录”进一步变成“可长期管理”。
